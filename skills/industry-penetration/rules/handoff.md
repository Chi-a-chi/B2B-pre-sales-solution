# 对话收尾机制

**触发词:用户说「转接新对话」**

自动输出转接文档,四部分:

## 1. 核心结论
本次对话产生的实质性判断和结果。

## 2. 悬置问题
未完成的待办、没讨论透的问题、下次需要继续的方向。

## 3. 上下文关键词
下次对话开场需要带入的背景(人名、项目名、已确认的前提、专有术语)。

## 4. 规则变更清单(三类全部列出)
- ① 建议并入 `~/.claude/CLAUDE.md` 或 `analysis-method.md` 的**通用规则**
- ② 建议进 `archive/project-context.md` 的**项目专属规则**
- ③ Claude 判断为一次性不保留的内容(用户有异议可当场捞回)
