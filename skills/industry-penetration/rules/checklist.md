# 行业洞察检查清单(通用)

> 本文件是 `analysis-method.md` 的配套检查工具。深度行业分析完成后,用本清单做完整性校验,发现遗漏就补。简单问题跳过,深度分析必须过一遍。
>
> 本文件由项目 CLAUDE.md 自动引用,无需手动上传。

---

## 模块一：推导链校验

检查分析是否形成完整因果链，而非并列模块堆砌。

### 标准推导链路
```
价值创造 → 参与方结构 → 流动路径 → 摩擦节点 → 完整机会图景 → 使用者角色定位 → 切入路径
```

### 检查项

- [ ] 每一步的输出是否是下一步的因果前提（不是并列信息）
- [ ] 是否有断链（某步结论和下步输入之间缺少逻辑连接）
- [ ] 是否有跳步（跳过中间环节直接给结论）
- [ ] 是否有结论无推导过程
- [ ] 是否有数据罗列不解释因果
- [ ] 完整机会图景是否先于使用者能力定位输出（顺序不能反）
- [ ] 机会图景是否因使用者能力局限而提前收窄

## 模块二：维度完整性校验（七层检查）

### 第一层：九流完整性

对每条流，检查是否描述了起点、终点、在哪里停留、停多久、沉淀在谁那里、谁有权动用。

- [ ] 信息流
- [ ] 合同流
- [ ] 物流（或服务流）
- [ ] 资金流
- [ ] 发票流
- [ ] 薪酬流
- [ ] 权益流
- [ ] 合规流
- [ ] 风险流

缺失任何一条要说明原因（该行业确实不涉及，而非遗漏）。

### 第二层：时间轴

- [ ] 各流之间是否有先后依赖关系
- [ ] 资金到账与票据匹配之间是否有时间差
- [ ] 时间差在哪里产生沉淀，沉淀资金归谁
- [ ] 合规申报周期是否与资金流动周期错位

### 第三层：业态完整性

- [ ] 是否有参与方同时扮演多个角色
- [ ] 是否有流动节点的归属关系不清晰
- [ ] 是否有环节的权责没有合同约定
- [ ] 是否存在联营/托管/撮合等特殊业态变体
- [ ] 对每个识别到的变体，是否重新检查了九流

### 第四层：竞争结构

- [ ] 上游供应商议价能力 → 是否造成资金流摩擦
- [ ] 下游买方议价能力 → 是否导致不对称风险
- [ ] 替代品或新进入者 → 摩擦是否在加剧，窗口是否收窄
- [ ] 现有竞争者布局 → 已有谁在做，留下什么空白
- [ ] 如果没有玩家：为什么没有？门槛还是市场太小？
- [ ] 平台类机会是否触发了网络效应检查（多边网络效应、先发优势、时机窗口）
- [ ] 现有摩擦节点是否已有解决方案 → 差异化空间在哪里

### 第五层：时机校验

- [ ] 当前生命周期阶段判断（必须说明判断依据）
  - 萌芽期 → 典型摩擦：信息不对称、合规缺位
  - 成长期 → 典型摩擦：资金错配、效率损耗
  - 成熟期 → 典型摩擦：权责不清、利润挤压
  - 衰退期 → 典型摩擦：结构性过剩、退出壁垒
- [ ] 识别到的摩擦是结构性还是过渡性
  - 结构性 = 长期存在，值得介入
  - 过渡性 = 市场会自然解决，窗口有限
- [ ] 介入是加速行业发展，还是在存量中抢份额
- [ ] 政策窗口：监管新规是否正在催化需求爆发
- [ ] 如果监管收紧，这个摩擦会变强还是变弱
- [ ] 如果技术替代发生，摩擦节点会消失还是转移

### 第六层：切入策略

- [ ] 摩擦最大承担者是头部还是长尾/弱势参与方
  - 弱势参与方痛更深，付费意愿更强，阻力更小
- [ ] 从弱势方切入后，是否有向上渗透的路径
- [ ] 现有头部在此节点是否存在过度服务（降维打击机会）
- [ ] 切入是否直接威胁头部核心利益 → 预判反制力度和进入成本

### 第七层：动态变化扫描

- [ ] **行业演变方向**：过去三年参与方结构在怎么变，谁在变强谁在变弱，趋势延续后摩擦在哪
- [ ] **同行竞争动态**：现有竞争者方案留下了什么空白，竞争者的核心假设是什么，如果这个假设是错的机会在哪里
- [ ] **政策与监管动向**：哪些灰色地带正在走向强制合规，政策收紧在哪个节点制造爆发式需求
- [ ] **技术变化的冲击**：哪项技术正在改变某个参与方的地位，被技术削弱的参与方会产生什么新摩擦，被技术赋能的参与方议价能力如何变化
- [ ] **外部压力催化**：哪个外部力量正在把摩擦从可忍受变成必须解决

## 模块三：输出质量校验

- [ ] 完整机会图景是否独立成段，先于能力定位输出
- [ ] 对每个核心摩擦节点，是否列出了所有可能的介入者类型（能力直接提供方 / 资源整合方 / 平台方 / 资本方 / 监管与行业组织 / 其他）
- [ ] 判断总结是否具体到节点和承担方（不是泛化判断）
- [ ] 每个结论是否有推导过程
- [ ] 是否存在跨行业经验直接套用未做适配
- [ ] 一句话洞察是否真的说清楚了（不超过60字）
